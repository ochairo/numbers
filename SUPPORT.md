# Support

## Before Opening an Issue

- Read the README and any package-specific docs first
- Search existing issues and pull requests for duplicates
- Prepare a minimal reproduction when reporting a bug

## Where to Ask for Help

- Reproducible bugs and actionable feature requests: GitHub Issues
- Contribution process and expectations: `CONTRIBUTING.md`
- Security vulnerabilities: `SECURITY.md`

## Good Reports Include

- package version
- Node.js and pnpm versions
- OS, browser, or runtime details
- minimal reproduction
- expected behavior and actual behavior
